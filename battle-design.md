# 战斗系统设计文档

> 本文档定义了 Keco Studio 回合制战斗系统的核心设计，精简为**元素系统**与**魔法值系统**两个核心模块。

---

## 一、属性系统

### 1.1 基础属性

| 属性 | 符号 | 说明 | 典型范围 |
|------|------|------|----------|
| 生命值 | HP | 生命归零则淘汰 | 1000 - 10000 |
| 攻击力 | ATK | 影响伤害计算 | 100 - 1000 |
| 防御力 | DEF | 减少受到的伤害 | 50 - 500 |
| 速度 | SPD | 决定行动顺序 | 50 - 200 |
| 魔法值 | MP | 释放技能消耗 | 50 - 500 |

### 1.2 伤害公式

```
实际伤害 = ATK × 技能倍率 × (ATK / (ATK + DEF)) × 元素反应倍率
```

### 1.3 速度与行动顺序

- 按 SPD 降序排列行动顺序
- SPD 越高，越先行动
- SPD 相同时随机决定

## 二、元素系统

### 2.1 元素类型

| 元素 | 符号 | 克谁 | 被谁克 |
|------|------|------|--------|
| 火 | 🔥 | 草 | 水 |
| 水 | 💧 | 火 | 雷 |
| 雷 | ⚡ | 水 | 草 |
| 草 | 🌿 | 雷 | 火 |
| 冰 | ❄️ | 草 | 火 |

### 2.2 元素强度

| 强度 | 持续回合 | 衰减率/回合 | 典型来源 |
|------|----------|-------------|----------|
| 弱 | 2 回合 | 10% | 普攻附着的元素 |
| 中 | 3 回合 | 15% | 普通技能 |
| 强 | 4 回合 | 20% | 怒气技能/高阶技能 |

### 2.3 元素衰减规则

- 每回合结束时，元素量 = 元素量 × (1 - 衰减率)
- 元素量低于 0.5 时，附着消失
- 回合结束显示剩余回合数

### 2.4 元素叠加规则

| 情况 | 处理方式 |
|------|----------|
| 同元素 | 保留较高元素量，持续时间取 max(原剩余, 新增) |
| 异元素 | 直接替换，不叠加 |
| 元素反应 | 消耗双方元素，产生对应效果 |

## 三、元素反应系统

### 3.1 反应类型

#### 增幅反应（伤害乘区）

| 反应 | 触发 | 倍率 | 消耗 |
|------|------|------|------|
| 蒸发 | 火 + 水 | × 2.0 | 各消耗 1 单位 |
| 融化 | 火 + 冰 | × 2.0 | 各消耗 1 单位 |

#### 剧变反应（额外伤害）

| 反应 | 触发 | 额外伤害 | 消耗 |
|------|------|----------|------|
| 感电 | 雷 + 水 | ATK × 0.8 | 各消耗 1 单位 |
| 超载 | 雷 + 火 | ATK × 1.0 | 各消耗 1 单位 |
| 燃烧 | 火 + 草 | ATK × 0.3/回合，持续 2 回合 | 各消耗 1 单位 |
| 冻结 | 水 + 冰 | 目标跳过下回合 | 各消耗 1 单位 |
| 激化 | 雷 + 草 | 后续雷/草伤害 +30%，持续 2 回合 | 各消耗 1 单位 |

### 3.2 反应触发流程

```
施放技能
    ↓
造成基础伤害（消耗 MP）
    ↓
为敌方附加元素
    ↓
检查敌方已有元素
    ↓（存在）
触发元素反应
    ↓
消耗双方元素
    ↓
执行反应效果
```

### 3.3 元素反应速查表

```
     │  火  │  水  │  雷  │  草  │  冰  │
─────┼──────┼──────┼──────┼──────┼──────┤
火   │  —   │ 蒸发 │ 超载 │ 燃烧 │ 融化 │
─────┼──────┼──────┼──────┼──────┼──────┤
水   │ 蒸发 │  —   │ 感电 │  —   │ 冻结 │
─────┼──────┼──────┼──────┼──────┼──────┤
雷   │ 超载 │ 感电 │  —   │ 激化 │  —   │
─────┼──────┼──────┼──────┼──────┼──────┤
草   │ 燃烧 │  —   │ 激化 │  —   │  —   │
─────┼──────┼──────┼──────┼──────┼──────┤
冰   │ 融化 │ 冻结 │  —   │  —   │  —   │
─────┴──────┴──────┴──────┴──────┴──────┘
```

## 四、魔法值系统

### 4.1 魔法值规则

| 参数 | 数值 | 说明 |
|------|------|------|
| 初始 MP | 100 | 战斗开始时 |
| MP 上限 | 100 | 可通过装备/天赋提升 |
| MP 恢复 | +10/回合 | 每回合结束恢复 |
| MP 恢复（击杀） | +20 | 击杀目标时额外恢复 |

### 4.2 魔法值不足处理

- 技能所需 MP > 当前 MP 时，技能不可用
- 普攻不消耗 MP
- 显示"魔力不足"提示

## 五、技能系统

### 5.1 技能数据结构

```typescript
interface Skill {
  id: string;                    // 技能唯一标识
  name: string;                  // 技能名称
  
  // 伤害参数
  power: number;                 // 技能倍率（相对于 ATK）
  mpCost: number;                // 魔法值消耗
  cooldown: number;              // 冷却回合
  maxCooldown: number;           // 最大冷却
  
  // 元素附着
  attachElement?: {
    element: 'fire' | 'water' | 'thunder' | 'grass' | 'ice';
    strength: 'weak' | 'medium' | 'strong';
    duration: number;            // 持续回合
  };
  
  // 特殊效果
  dot?: {                         // 持续伤害
    damage: number;               // 每回合伤害倍率
    duration: number;
  };
  
  crowdControl?: {                // 控制效果
    type: 'freeze';
    duration: number;
  };
  
  description: string;             // 描述
}
```

### 5.2 技能示例

| 技能名 | MP 消耗 | 冷却 | 伤害 | 效果 |
|--------|---------|------|------|------|
| **火球术** | 20 | 5 | 1.5×ATK | 附加火元素（强·3回合）；灼烧 DOT 0.3×ATK/回合 |
| **水波术** | 15 | 3 | 1.3×ATK | 附加水元素（中·2回合） |
| **雷鸣击** | 25 | 4 | 1.2×ATK | 附加雷元素（强·3回合）；若目标有水，触发感电 |
| **冰霜术** | 15 | 4 | 1.0×ATK | 附加冰元素（中·2回合）；若目标有水，触发冻结 |
| **荆棘术** | 20 | 3 | 1.1×ATK | 附加草元素（中·2回合）；若目标有雷，触发激化 |
| **烈焰斩** | 40 | 6 | 2.5×ATK | 附加火元素（强·3回合）；若目标有水，触发蒸发 |
| **普攻** | 0 | 0 | 1.0×ATK | 无附加效果 |

### 5.3 技能冷却规则

| 情况 | 处理 |
|------|------|
| 技能释放后 | 冷却 = maxCooldown |
| 每回合结束 | 冷却 - 1（最低为 0） |
| 冷却中 | 技能不可用，显示剩余回合 |

## 六、战斗流程

### 6.1 回合流程

```
回合 N 开始
│
├── 阶段 1：回合开始处理
│   ├── 结算 DOT（灼烧等持续伤害）
│   ├── 减少元素附着回合（< 1 时消失）
│   └── 减少控制状态
│
├── 阶段 2：速度排序
│   └── 按 SPD 降序排列行动顺序
│
├── 阶段 3：行动阶段
│   ├── 每个角色依次行动
│   │   ├── 检查是否被控制（冻结则跳过）
│   │   ├── 选择技能
│   │   ├── 检查 MP 是否足够
│   │   ├── 扣除 MP
│   │   ├── 造成伤害
│   │   ├── 触发元素反应
│   │   └── 减少技能冷却
│   └── 回合结束处理
│
├── 阶段 4：回合结束
│   ├── MP 恢复 +10
│   ├── 减少技能冷却
│   └── 检查战斗结束条件（HP <= 0）
│
回合 N 结束
```

### 6.2 元素反应触发示例

```
回合 2：
[玩家] 使用「火球术」消耗 20 MP，对 [Boss] 造成 45 伤害
[元素] [Boss] 获得「火·强」（剩余 3 回合）
[元素] [Boss] 获得「灼烧」（剩余 2 回合）
    └─ 每回合损失 9 HP

回合 3：
[玩家] 使用「水波术」消耗 15 MP，对 [Boss] 造成 39 伤害
[元素] 「火·强」+「水·中」触发「蒸发反应」！
    └─ 蒸发伤害：45 × 2.0 = 90
    └─「火·强」已消失
    └─「水·中」残留 2 回合

[Boss] 被「灼烧」损失 9 HP

回合结束：MP +10
```

## 八、数据结构

### 8.1 类型定义

```typescript
// 元素类型
type Element = 'fire' | 'water' | 'thunder' | 'grass' | 'ice';

// 元素强度
type ElementStrength = 'weak' | 'medium' | 'strong';

// 元素附着状态
interface ElementAttachment {
  element: Element;
  strength: ElementStrength;
  amount: number;        // 当前元素量
  remainingTurns: number; // 剩余回合
}

// 战斗单位
interface Combatant {
  id: string;
  name: string;
  hp: number;
  maxHp: number;
  atk: number;
  def: number;
  spd: number;
  mp: number;
  maxMp: number;
  element: ElementAttachment | null;  // 当前元素附着
  dot: {                               // 持续伤害
    damage: number;
    remainingTurns: number;
  } | null;
  control: {                           // 控制状态
    type: 'freeze';
    remainingTurns: number;
  } | null;
  skillCooldowns: Record<string, number>;  // 技能冷却
}

// 技能
interface Skill {
  id: string;
  name: string;
  power: number;
  mpCost: number;
  cooldown: number;
  attachElement?: {
    element: Element;
    strength: ElementStrength;
    duration: number;
  };
  dot?: {
    damage: number;
    duration: number;
  };
  crowdControl?: {
    type: 'freeze';
    duration: number;
  };
  description: string;
}

// 战斗日志条目
interface BattleLogEntry {
  turn: number;
  actor: string;
  action: string;
  damage?: number;
  elementReaction?: string;
  mpCost?: number;
  otherEffects?: string[];
}
```
